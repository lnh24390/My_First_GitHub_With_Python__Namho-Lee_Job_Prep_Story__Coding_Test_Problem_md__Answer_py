# 코딩테스트 개요
코딩 테스트(Coding Test)는 소프트웨어 개발 직군 채용 과정에서 지원자의 문제 해결 능력, 알고리즘 이해도, 코드 구현 역량을 평가하기 위한 표준화된 평가 방식이다. 특히 국내외 IT 기업에서 필수적인 전형으로 자리 잡고 있다.

## 1. 목적 및 평가 요소

코딩 테스트는 단순한 문법 숙련도를 넘어 다음과 같은 역량을 종합적으로 검증한다.  
  
문제 해결 능력: 주어진 문제를 분석하고 해결 전략을 수립하는 능력  
알고리즘 및 자료구조 이해: 시간/공간 복잡도를 고려한 최적화 설계  
구현 능력: 논리적 사고를 코드로 정확히 변환하는 능력  
디버깅 능력: 오류를 식별하고 수정하는 역량  
## 2. 시험 유형
### (1) 온라인 자동 채점형
플랫폼: 백준 온라인 저지, 프로그래머스, LeetCode  
특징:  
제한 시간 내 문제 풀이  
입력/출력 기반 자동 채점  
다수의 테스트 케이스로 정확성 검증  
### (2) 라이브 코딩 인터뷰
화상 또는 대면 환경에서 진행  
문제 해결 과정(사고 과정)을 중점적으로 평가  
코드 품질 및 커뮤니케이션 능력 포함  
### (3) 과제형 테스트
일정 기간 내 프로젝트 또는 문제 세트 제출  
실제 개발 역량(설계, 코드 구조 등) 평가  
## 3. 주요 출제 범위
  
코딩 테스트는 특정 프로그래밍 언어보다 컴퓨터 과학 기반 개념을 중심으로 출제된다.  

자료구조  
배열, 스택, 큐, 해시맵, 트리, 그래프  
알고리즘  
정렬, 탐색(이진 탐색), DFS/BFS  
고급 기법  
동적 계획법(DP)  
그리디 알고리즘  
백트래킹  
수학/기타  
조합론, 문자열 처리, 비트 연산  
## 4. 평가 기준
  
기업마다 다르지만 일반적으로 다음 요소가 반영된다.  

정확성 (Correctness): 모든 테스트 케이스 통과 여부  
효율성 (Efficiency): 시간 복잡도 O(n), O(log n) 등  
코드 품질 (Readability): 가독성, 변수명, 구조  
문제 해결 접근 방식 (Approach)  
## 5. 준비 전략
  
효율적인 대비를 위해서는 체계적인 접근이 필요하다.  
  
기초 알고리즘 학습  
자료구조 및 시간복잡도 개념 숙지  
문제 풀이 반복  
하루 1~2문제 꾸준히 풀이  
플랫폼 활용  
백준 온라인 저지: 난이도별 체계적 학습  
프로그래머스: 기업 기출 기반 문제  
언어 선택  
Python, Java, C++ 중 하나를 집중적으로 숙련  
## 6. 특징 및 최근 트렌드
단순 구현 문제 → 복합 문제(자료구조 + 알고리즘 결합) 증가  
시간 제한이 엄격해져 최적화 능력 중요성 증가  
일부 기업은 SQL, 시스템 설계 문제 병행  
## 핵심 요약
  
코딩 테스트는 단순한 코딩 능력이 아니라  
  
문제를 구조적으로 분석하고, 효율적인 알고리즘으로 해결하며, 이를 정확한 코드로 구현하는 종합 능력 평가 과정이다.
